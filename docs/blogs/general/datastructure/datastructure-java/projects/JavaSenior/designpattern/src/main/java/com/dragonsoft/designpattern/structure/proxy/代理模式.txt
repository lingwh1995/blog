代理模式:
	通过为被代理对象创建一个代理对象,实现对被代理对象的访问控制,可以保护被代理对象
1.动态代理、静态代理和cglib代理:
	1>.当需要对返回值进行处理时,优先考虑静态代理,因为静态代理可以不用强制转换就得到明确的返回值类型,而动态代理需要将Object类型的返回值转换为具体的类型
	2>.当需要在方法执行前后进行增强时三种都可以使用
2.代理模式要实现的目的?
	1>.保护和增强被代理对象
	2>.实现访问控制,即不把真实的对象暴露给外部,而是把代理对象暴露给外部
	3>.提高了被代理类的安全性
3.静态代理和动态代理的区别?
	1>.静态代理在编译期就生成了代理类	
	2>.动态代理是在运行时通过反射动态创建代理类
